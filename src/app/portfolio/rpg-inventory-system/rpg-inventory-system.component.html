Overview:

This project aims to reproduce an inventory system similar to one you'd see in RPG games such as Baldur's Gate 3, Divinity Original Sin 2 or Pillars of Eternity 2. Characters have their own inventory, where each item takes up one cell in the grid. The player can decide in which cell what item should be and perform common actions, such as splitting, combining or swapping items.

Features:

Items can have various features, such as being equippable. For example, if a player character equips a Plate Armor item, the game would update the character's model in game world and UI to reflect that change.

Some items can also be containers, and as such have their own inventory. This mean you could create a bag item that the player could open and put some other items inside.

Items in the project are created in such a way to make it easy to add new functionality. You could create an item feature that, for example, modifies the character's strength for as long as they have the item in their inventory, or equipped.

Multiplayer:

Item instances held by actors in-game are fully replicated. Any changes made to the instance will be propagated from the server to all connected clients. Items, which are referenced by item instances, are also replicated, which allows them to be modified and created on runtime.

You can find the code on https://github.com/Ilethas/RPG-Inventory-System. The repository contains only the source code, as the project contains paid assets such as models and animations.

Items can be easily designed in-editor using Data Assets. Characters' inventories keep pointers to Item Instance actors that reference Item objects defined as Data Assets. Both support network replication.
